技術棧背景交代

前後端技術

  • 前端框架: Vue.js 3（畫面渲染）
  • 應用框架: Nuxt.js（路由、建構）
  • 後端框架: FastAPI（API、商業邏輯）

架構模式

  • 開發環境: 前後端分離開發

    • 前端: npm run dev (Vite dev server)
    • 後端: uvicorn main:app --reload

  • 生產環境: 靜態前端 + API 服務器一體化部署

    • 前端編譯: npm run generate → 靜態文件
    • 部署方式: FastAPI 統一託管靜態文件與 API
    • 優點: 單一服務、無需額外靜態文件服務器、避免 CORS 問題

環境建置步驟

建構開發環境

  1. 建立虛擬環境 : uv init
  2. 安裝 fastapi 跟 fastapi 的伺服器 uvicorn : uv add fastapi uvicorn
  3. 寫一點測試的東西在 main.py
  4. 運行伺服器進行測試 uv run uvicorn main:app --reload
  5. 確認 fastAPI 運行成功後，開始部屬Nuxt.js
  6. 根據文件安裝 Nuxt.js npm create nuxt@latest <project-name>
  7. 安裝過程會問一些問題，請根據自身專案的特性來確認
  8. 安裝完成後，開啟 Nuxt 的開發伺服器來確認是否可運行 npm ruv dev

測試生產環境

  1. 需要先在 nuxt.config.ts 裡設置 ssr: false >> Deployment#Static-Hosting
  2. 使用 npm run generate 產生靜態檔案，需要注意 Nuxt 產生的靜態檔案會在哪個目錄 >> 基於爬蟲的預渲染
  3. 並根據這份文件 FastAPI Doc # Static Files，將 Nuxt 產生的靜態檔案的目錄「掛載(Mounting)」進 FastAPI。
  4. 運行 FastAPI 伺服器後，會在這個網址出現 Nuxt 的成功畫面。

問題集合

  1. Nuxt 在 SSR 渲染模式下，沒有生成 FastAPI 可用的 HTML 檔。

    • 初始問題

      • Nuxt 的渲染模式預設為 ==[^1]SSR（伺服器端渲染）==。
      • 需要使用 Node.js 執行 JavaScript 來渲染 Vue 元件。
      • 由於 FastAPI 是 Python 框架，不支援 Nuxt 的 SSR 模式。

    • 解決辦法：

      • 在 nuxt.config.ts 中關閉 SSR 模式。
      • 改用 SSG（靜態生成）或 SPA（單頁應用）模式。
      • 這樣 FastAPI 就能託管打包後的靜態檔案。

  2. 關閉 SSR 模式之後，使用 build 依然不會生成 HTML，需要把指令改成 generate 。

    • 修改成 ssr: false後：
      • npm run build 會生成 SPA (Single Page Application)
        模式的文件
      • npm run generate 會生成 靜態 HTML + SPA 的混合模式
    • 兩個指令的差異：

      • npm run build

        • 用於 SSR (Server-Side Rendering) 模式
        • 生成需要 Node.js 服務器運行的應用
        • 輸出包含服務器端代碼和客戶端代碼
        • 需要用 node .output/server/index.mjs 啟動服務器
        • 適合動態內容、需要服務器端渲染的應用

      • npm run generate

        • 用於 靜態站點生成 (Static Site Generation, SSG)
        • 預渲染所有路由，生成純靜態文件（HTML、CSS、JS）
        • 輸出在 .output/public 目錄
        • 可以直接託管在任何靜態文件服務器（Nginx、Apache、FastAPI、CDN 等）
        • 因為設定了 ssr: false 且要交給 FastAPI 託管

  3. HTML 中的資源（JS / CSS）路徑與 FastAPI 掛載的路徑不符合

    • 說明
      • 在 FastAPI 掛載 Nuxt 生成的靜態檔案時，我是掛載在 /statci 的網址路徑下：
      • 而 ==[^2]Nuxt 生成靜態檔案裡，預設生成的資源路徑==是絕對路徑：/_nuxt/....
      • 所以瀏覽器會像 FastAPI 發送這個請求：https://example.com/_nuxt/....
      • 但是 FastAPI 裡，我們掛載了 /static 這個網址路徑，所以它只處理 /static 開頭的這種請求：https://example.com/static/_nuxt/....

[^1]: [HTML 渲染模式：SSR、SSG、SPA]。
[^2]: [關於baseURL 路徑設定]。
